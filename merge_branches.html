Create and Merge Branches

CHEATSHEET:

WHAT IS A BRANCH?, CREATE A NEW BRANCH, MERGING BRANCHES

+ What is a Branch?

Git stores data as snapshots and commits are essentially pointers to each progressive commit. You can think of a branch as the context you're working in. And switching branches means you're switching contexts.

A Git 'branch' is nearly synonymous to a 'head', which means that every branch represents one 'head' and can represent an entire ancestoral history of commits preceding that 'head' in an isolated development environment.

You can also think of a 'head' as the 'current branch'. And when you switch branches with 'git checkout', the 'head' now points to the tip of the new branch. Each commit stores the latest change to a branch, which allows you to go back and forth in a project, within the context of a branch.

Say for instance you're working with a group of programmers. Each of your team mates has been assigned a new feature. To work with the same 'repo', you create a dedicated branch and then switch into it, allowing you to work independently before merging your work to the 'master' branch.

You can always find out which branch you're on, by typing 'git branch', which lists the available branches. The starred branch is the one you're currently on.

$    git branch
master
* new_feature
+ Create a New Branch

You can create a new branch by running 'git branch test'. Now go ahead and run 'git branch' which will note the active branch with an asterick.

$    git branch test
$    git branch
* master
test

We've created a new branch 'test', but we are still on the 'master' branch. Let's switch to the new branch, 'test'. That's easy, simply run 'git checkout test'. And then run 'git branch' to see which branch we're on.

$    git checkout test
$    git branch
master
* test

Next create a file called 'testing.rb' in your project, then stage it by running 'git add testing.rb', and commit your change. Now switch back to the master branch, with 'git checkout master'. Finally list the contents of this branch by running 'ls'.

$    git add testing.rb
$    git commit -m "added a file testing.rb"
$    git checkout master
$    ls

Remember, 'ls' will list what is in the current directory of the 'master' branch; however anything added to the test branch will not be listed.

Now you're in the 'master' branch but you haven't merged branches yet. When you're ready to merge 'test' you can do so by running 'git merge test'.

+ Merging Branches

To Merge a branch you first need to start in the 'master' branch, then run the 'git merge' command. This will join two development histories together into a single branch.

Then when the two branches have merged, it's a common workflow to delete the working branch, in this case you would be deleting the 'test' branch.

$    git checkout master
$    git merge test
$    git branch -d test

When there is a conflict, git will prevent the branches from merging. You can then run 'git status' which will show you the files that need to be resolved.

$    git status

Typically small features are used for 'fast-foward' merges and larger ones are called 3-way merges that include larger feature branches.

For more info, Pro-Git does a great job of describing the basics of branching and merging.

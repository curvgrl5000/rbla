COMMAND LINE BASICS

+ What is the Command Line?

Many of us are used to using GUIS (Graphical User Interfaces), but learning how to use the CLI ( Command Line Interface ), is essential for programming. Common names include the Shell, Bash, CLI, Terminal and Console. But exactly what is the Command Line? At it’s core, it’s simply the place where you type commands.
Wikipedia defines the Terminal this way:

“A command-line interface (CLI) is a means of interaction with a computer program where the user (or client) issues commands to the program in the form of successive lines of text (command lines). A command-line interface is one type of text-based user interface.”

On a Mac, launch the Command Line from the utilities folder inside of your main Applications folder. Apple Mac OS X[4] and many Linux distributions have the Bash implementation of the Unix shell. 'Bash' by the way, is an acronym for 'Bourne-Again Shell', a pun on Stephen Bourne, who developed it at Bell Labs.

On MS Windows users use the Command Prompt or the PowerShell. Android uses a Unix shell derived from Ash with commands from the separate toolbox.

We will be concerning ourselves with the Mac side of things.

+ The syntax of the Command Line

Most commands have three parts and start with the '$' symbol. The Utility or Command Name, The Flags or Options and The Argument(s).

$    ls -1 ~/Desktop

Let's break each part down:

$ is the common symbol for SELECT, which indicates that you’re ready to type commands into the command line. You don’t have to type this, it’s inherited with the prompt.

ls is the UTILITY. Generally known as commands as they are indicating what you want the computer to do. In this case 'ls' is the command to list the contents of directories.

-1 is the FLAG or OPTION. This changes or alters how a utility may operate slightly. Usually the default works just fine, but flags allow you to modify them to your liking. They start with one or two dashes. In this case, the option '-1' modifies the utility 'ls', to list one file per line.

~/Desktop is the ARGUMENT to the utility. Arguments are used to define exactly what the utility might need to know for certain actions, and there is no clear default setting.

Here, the argument instructs the utility to list the contents of the subdirectory Desktop, nested under your HOME directory. That squiggly thing '~', is the tilde, which defaults to the value of a users HOME shell variable.

To get a list of all commands, type the following:

$    man command

FYI: Man means manual.

+ Most Common Commands

Remember guys, you're ready to go when you see the '$' in the terminal. You're now ready to code. Here are some of the most commonly used commands:

cd is shorthand for Change Directory and allows you to go to another directory. Remember you have to define this utility with a path. For instance: cd ~/my_machine/new_path literally means change the directory with this argument; which is a path.

cd .. is shorthand for Change Directory .. and allows you to go up a directory vs. down a directory.

exit is shorthand for Exit and literally exits a terminal window.

irb is shorthand for Interactive Ruby, which you have to have installed. Now watch for the prompt! When launching irb, the prompt will change from '$' to '>'. That means you have to exit irb with the buttons 'control d', to get back to the former context.

Ctrl-D is shorthand for Control D. 'D' is the default value that tells the terminal to register an EOF or 'End of File' on standard input, which the terminal interprets as a desire to exit.

ls is shorthand for Lists which literally lists what is in the current directory.

man -k "topic" is shorthand for Manual Lookup Topic, and looks up and displays what you're looking for in quotes.

mkdir is shorthand for Make Directory, and allows you to make a folder. You must define this utility with an argument, which in this case is the name of the folder.

pwd is shorthand for Print Working Directory, which gives you the full path of where you are.

rails new app_name is shorthand for Rails New Application Name, and will create the entire directory structure of a new rails application. 'rails' loads the rails Gem, 'new' is the command that creates a new rails app, and 'app_name' is just the placeholder for the name you'll be giving your app.

rails s is shorthand for Rails Server, which launches a local server. Once you've typed 'rails s' in the commanline, type: localhost:3000 in a browser's address bar, which will render a local version of your work in the browser.
